1.2时间戳和计时器

  • time.time() 返回自纪元以来的秒数，记录sleep

  • time.perf_counter() 随意选取一个时间点，记录现在时间到该时间点的间隔秒数，记录sleep

  • time.process_time() 随意选取一个时间点，记录现在时间到该时间点的间隔秒数，不记录sleep
    perf_counter() 精度比time()更高一些，但time()在大多数情况下能完美符合使用要求。

3.collections——容器数据类型

3.1 namedtuple——具名元组

  1. 不可变性：具名元组的值一旦被赋值后，不能被修改，类似于元组。
  2. 可读性：可以通过属性名访问元素，而不是通过索引，使得代码更加易读。
  3. 轻量级：具名元组的内存占用比普通类对象更小，性能接近普通元组。
  4. 兼容性：具名元组可以像普通元组一样使用，支持解包、迭代等操作。
  5. 可以调用属性，具有元组的性质，是元组的子类。

定义方法

namedtuple(typename, field_names, *, rename=False, defaults=None, module=None)

  1. typename：具名元组的类名，通常是一个字符串。

  2. field_names：字段名称的序列（如列表、元组）或一个空格分隔的字符串。字段名称必须是有效的 Python 标识符。

  3. rename（可选）：布尔值。如果为 True，则会自动重命名不符合 Python 标识符规则的字段名称。

  4. defaults（可选）：一个可迭代对象，为字段提供默认值。从 Python 3.7 开始支持。

  5. module（可选）：指定具名元组所属的模块名称。

3.3 deque——双向队列

  1. 列表访问数据非常快速

  2. 插入和删除操作非常慢一通过移动元素位置来实现

  3. 特别是insert(0, V)和pop(0),在列表开始进行的插入和删除操作

  4. 双向队列可以方便的在队列两边高效、快速的增加和删除元素

    deque([‘c’, ‘d’, ‘e’])

    deque([‘a’, ‘b’, ‘c’, ‘d’, ‘e’, ‘f’, ‘g’])

    deque([‘b’, ‘c’, ‘d’, ‘e’, ‘f’])
